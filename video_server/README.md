# QoD Video Test Server (HLS/DASH)

This folder provides a reproducible HLS/DASH video server for QoD testing.

## What you get

- Dockerized `nginx` server on port `8080`
- Automatic adaptive HLS (`.m3u8`) ladder up to 4K and DASH (`.mpd`) packaging using `ffmpeg`
- A simple browser test player at `/player.html`

HLS ladder variants generated by default:

- 2160p (4K)
- 1080p
- 720p
- 480p

## Folder layout

- `data/input/source.mp4` - optional input video (bring your own)
- `output/hls` - generated HLS files
- `output/dash` - generated DASH files

If `data/input/source.mp4` is missing, a synthetic 3-minute sample is generated automatically.

## Run locally or on AWS EC2

1. Place your test file at `data/input/source.mp4` (optional).
2. Start the server:

```bash
docker compose up --build
```

3. Open:
   - Player page: `http://<server-ip>:8080/player.html`
   - HLS manifest: `http://<server-ip>:8080/media/hls/stream.m3u8`
   - DASH manifest: `http://<server-ip>:8080/media/dash/stream.mpd`

## QoE KPI capture (baseline vs QoD)

The player includes run controls and export:

- `Start Run`: starts KPI tracking
- `Load`: loads current stream URL
- `End Run`: closes the run and stores it in browser memory
- `Export JSON` / `Export CSV`: downloads all completed runs

Captured KPIs per run:

- `startupDelayMs`
- `rebufferCount`
- `rebufferTotalMs`
- `watchTimeSeconds`
- `playedSeconds`
- `bufferingRatio`

Suggested process:

1. Run 10+ baseline plays (without QoD), exporting `qoe_runs_baseline.csv`.
2. Enable QoD for your AWS Elastic IP and run the same test again.
3. Export `qoe_runs_qod.csv` and compare median/p95 startup and buffering metrics.

## AWS tips for QoD testing

- Use an EC2 instance with a fixed Elastic IP.
- Allow inbound TCP `8080` (or put behind `443` reverse proxy).
- Configure your QoD session service IP as this Elastic IP.
- Keep the same video and test script for baseline vs QoD runs.

## Useful environment overrides

In `docker-compose.yml`:

- `INPUT_FILE` (default: `/data/input/source.mp4`)
- `HLS_NAME` (default: `stream`)
- `DASH_NAME` (default: `stream`)
- `SEGMENT_SECONDS` (default: `4`)
